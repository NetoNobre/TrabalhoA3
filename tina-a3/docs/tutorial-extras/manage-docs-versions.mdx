## 06/04 - Panorama Geral do Projeto: 2º Entrega

# Escolher e justificar o modelo de processo de software mais adequado às necessidades do projeto:

A escolha do modelo cascata se deu pois pela sua estrutura simples o modelo cascata é fácil de entender e usar. Ele segue uma abordagem linear e sequencial, o que simplifica o processo de desenvolvimento, e gerenciamento de projetos

Além disso o uso do método cria uma documentação abrangente com cada fase tendo seus próprios documentos e artefatos associados, o modelo cascata tende a gerar uma documentação detalhada ao longo do ciclo de vida do projeto. Isso pode ser útil para referência futura, manutenção e treinamento de equipe.

Em resumo facilita o controle de qualidade com cada fase tendo suas próprias etapas de verificação e validação, o controle de qualidade pode ser integrado em todo o ciclo de vida do projeto. Isso pode resultar em produtos finais mais consistentes e de maior qualidade.

# Serão feitas Sprints com tempo médio de 5 a 8 dias para abordagem de tais temas:

Autenticação e Gerenciamento de Usuários:

- Implementar sistema de login e autenticação.

- Permitir que os usuários criem e gerenciem seus perfis.

- Cadastro e Gerenciamento de Ferramentas:

- Permitir a categorização das ferramentas por tipo, categoria, e finalidade.

- Desenvolver funcionalidade para pesquisar e filtrar ferramentas com base em diferentes critérios.

Integração de Ferramentas Externas:

- Integrar ferramentas populares de engenharia de software, como Git, Jira, Jenkins, etc.

- Sincronizar dados relevantes entre a plataforma e as ferramentas externas.

Recursos Avançados de Pesquisa e Análise:

- Implementar um mecanismo de pesquisa avançado para encontrar ferramentas com base em critérios específicos.

Suporte a Metodologias Ágeis:

- Integrar ferramentas de framework ágil, como quadros Kanban.

Interface do Usuário e Experiência do Usuário (UI/UX):

- Design de uma interface intuitiva e fácil de usar para navegar e interagir com a plataforma.

- Melhorar a experiência do usuário com feedback instantâneo, animações, etc.

- Realizar testes de usabilidade para identificar e corrigir quaisquer problemas de interface ou fluxo de trabalho.

Segurança e Desempenho:

- Realizar auditorias de segurança e correções de vulnerabilidades identificadas.

- Realizar otimizações de desempenho para garantir que a plataforma seja responsiva e escalável.

## 08/04 - Revisão e Verificação

Como todos os requisitos e verificações tinham sido atendidos, essa Sprint utilizamos para verificar se todos os requisitos tinham sido atendidos e se precisaria acrescentar mais ou remover algum requisito. Portanto, decidimos deixar o requisito de Banco de dados opcional, então, não utilizamos, porém pode ser usado e bem combinado.

Decidimos também remover o requisito de "Comentários e feedbacks", porque o foco da plataforma é mostrar o catálogo com informações úteis para ferramentas. Portanto, ficaria algo irrelevante para a Bifrost.
